Getting Started

Up and running in under 5 minutes.

One command to scaffold. Install. Build. Deploy. That's it.

1. Create a new project

The scaffolder creates a complete project with schema, example content, and AI configuration.

bash
$ npx @webhouse/cms init my-site

my-site/
  cms.config.ts          # Collection + field definitions
  package.json           # Dependencies
  .env                   # AI provider keys
  content/
    posts/
      hello-world.json   # Example document

2. Install and run

Start the dev server and admin UI. Content changes are live-reloaded.

bash
$ cd my-site
$ npm install
$ npx cms dev

✓ Admin UI → http://localhost:3000/admin
✓ Dev server → http://localhost:3000
✓ Watching for changes...

3. Build for production

The build pipeline generates optimized static HTML with SEO, sitemaps, RSS, and AI discovery files.

bash
$ npx cms build

✓ 12 pages rendered
✓ sitemap.xml generated
✓ robots.txt generated
✓ llms.txt + llms-full.txt generated
✓ feed.xml RSS feed generated
✓ Per-page .md files for AI
→ Build complete in 1.2s

What you get

Every build generates a complete static site with:

  • HTML pages for all published documents
  • sitemap.xml for search engines (with hreflang for i18n)
  • robots.txt with AI-optimized crawler rules
  • llms.txt and llms-full.txt for AI discovery
  • feed.xml RSS feed
  • Per-page .md files for AI consumption
  • JSON-LD structured data on every page
  • OG metadata for social sharing
